## ALM Corp Audit Framework

*Source: almcorp.com/blog — considered one of the most analytically rigorous Google Ads publications*

### Google Search Ads Audit 2026: Key Methodology

ALM Corp argues that traditional 2020-era audit frameworks are obsolete. The platform has fundamentally changed: exact match no longer means exact, manual bidding is deprecated, and campaign types are bundled by default.

**Marginal CPA Analysis:**
The core analytical innovation is identifying **diminishing returns thresholds** — the point where additional spend produces negative marginal returns. Key insight: "When ROAS is acceptable at a blended level but marginal returns have turned negative at the last $50,000 of spend — the dashboard does not tell you that." Modern audits must identify where "spend crosses the diminishing returns threshold," distinguishing between blended metrics that hide expensive performance at the margin.

**Audit Framework Pillars:**
1. Marginal returns analysis — not blended ROAS but incremental CPA at each spend tier
2. Conversion action audit — are the right conversions being tracked and valued?
3. Campaign structure evaluation — does the structure match the current platform's capabilities?
4. Match type and query analysis — with match types fundamentally redefined
5. Bid strategy alignment — is the strategy appropriate for conversion volume?

### AI Max for Search: 250-Campaign Study

ALM Corp independently analyzed 250+ real campaigns running AI Max:
- Median revenue uplift: **+13%**
- Median CPA increase: **+16%**
- ROAS range: **+42% to -35%** (extremely high variance)
- Analyst conclusion: "Turning on AI Max is essentially a coin toss: you may see a lift, but efficiency likely won't follow"
- Google's own 14% benchmark notably **excludes retail** — a significant omission for ecommerce

**Implication for BrightMatter:** AI Max adoption is a coin flip without account-specific context. Pattern detection across 500 accounts could identify the account characteristics that predict whether AI Max helps or hurts.

*Source: 9,199 accounts, 24,702 campaigns*

| Structure Type | Description | ROAS | CVR | CPA |
|---|---|---|---|---|
| Starter | 1 campaign / 1 asset group | Baseline | Baseline | Baseline |
| **Focused** | **Multiple campaigns / 1 asset group** | **Best** | **2nd best** | **Best** |
| Conversion Hungry | 1 campaign / multiple asset groups | Mixed | Mixed | Mixed |
| Mixed | Multiple campaigns / multiple asset groups | Variable | Variable | Variable |

### Budget Allocation Findings

| PMax Budget Share | ROAS | CVR | CPA |
|---|---|---|---|
| >50% of total budget | 652.03% (strongest) | Mixed | Mixed |
| 10–25% of total budget | Moderate | Best | Best |

### Key Structural Insights

- PMax campaigns struggle when paired with other campaign types (cannibalization)
- Multiple campaigns with one asset group consistently outperformed

## Agent Architecture Patterns

### PPC.io — Specialized Agent Architecture

PPC.io's approach rejects general-purpose AI for Google Ads management in favor of **specialized, single-purpose agents**, each built for one task.

**Core Architecture:**
- **Task-specific agents** rather than one "do-everything" AI
- Each agent uses a specialized data processing model optimized for its task
- Built-in data validation and verification to prevent hallucinations
- Industry-specific benchmarking baked into each agent

**Agent Library:**
1. **Account Audit Agent** — anomaly detection algorithms for ad spend pattern recognition, prioritized action plans
2. **Client Reporting Agent** — contextualizes performance against goals, historical trends, and industry benchmarks
3. **Negative Keyword Agent** — **proprietary N-gram pattern detection** to identify hidden keyword patterns; uses statistical validation to ensure recommended negatives won't impact converting traffic; categorizes by confidence level and match type
4. **Landing Page Performance Agent** — evaluates message match between ads and landing pages; industry-specific landing page quality standards
5. **Keyword Opportunity Agent** — competitive intelligence, gap analysis, business relevance scoring

**Why General AI Fails for PPC (per PPC.io's testing):**
1. Limited context windows — can only process fraction of account data, missing cross-campaign patterns
2. Inconsistent outputs — same prompt, same account, 100 different answers
3. Prompt dependency — expertise in prompt engineering required just to get usable answers
4. No verification mechanisms — confidently suggests policy-violating changes
5. Missing benchmarks — can't contextualize whether 2% CTR is good or terrible for a specific industry

**Key Innovation for BrightMatter:** The N-gram pattern detection for negative keyword management is directly relevant to BrightMatter's query analysis across 500 accounts. Statistical validation before recommending negatives (ensuring they won't block converting traffic) is a critical safety mechanism.

### Roadway AI — Google Ads AI Agent Framework

Roadway AI provides a structured framework for building AI agents that monitor, plan, and execute Google Ads optimizations.

**Three-Layer Architecture:**

1. **Monitoring Layer**
   - Pulls performance data on schedule
   - Joins with cross-channel attribution data
   - Checks all funnel metrics against targets
   - Surfaces ranked deviations (not full reports)

2. **Planning Layer**
   - Reasons about what to do given monitoring findings
   - Conversational: agent proposes, human pushes back
   - Memory system logs past changes, outcomes, experiments, seasonal patterns
   - Accuracy improves over 4–6 weeks of accumulated history

3. **Action Layer**
   - Generates change manifest with reasoning per line item
   - Human approves/rejects each change
   - Executes via Google Ads API mutate operations
   - Operations: keyword pauses, bid adjustments, budget shifts, negative additions, new keywords from search terms

**Attribution Foundation (Critical Insight):**
Every platform overcounts its own contribution. Roadway insists the agent must reason from a **unified attribution model** measuring all channels the same way — not Google's self-serving attribution windows. Without cross-channel attribution, "your agent optimizes for whatever proxy metric you have given Google, and those proxies are not your business goal."

**Configuration Architecture:**
- **Goal metric** — single number the agent optimizes for (paid customers, revenue, qualified pipeline)
- **Funnel metrics** — chain of events between click and goal (CTR → landing page CVR → trial signup → activation → paid conversion)
- **Guardrail metrics** — constraints (max CPL, minimum conversion volume before structural changes, impression share floor on brand terms)
- **Memory** — structured log of changes, outcomes, experiments, seasonal patterns; updated after each run, passed as context every time

**Skills (Markdown Files for Agent Context):**
- Account strategy, keyword strategy, budget allocation rules, negative keyword management, bid management rules, competitive context

**Key Insight for BrightMatter:** The three-layer architecture (monitor → plan → act) with memory persistence and guardrail metrics maps directly to how BrightMatter should operate across 500 accounts. The unified attribution requirement means BrightMatter needs to either integrate cross-channel data or explicitly flag when its recommendations are based on Google-only attribution (which systematically overcounts Google's contribution). The 30 conversions/month minimum threshold for reliable pattern detection aligns with expert consensus.

**Early Results:** Customers report **25% CAC reductions** and up to **10 hours/week saved**.
